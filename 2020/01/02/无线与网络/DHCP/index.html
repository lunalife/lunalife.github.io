基础

DHCP：Dynamic Host Configuration Protocol。

DHCP服务器帮助我们自动的将网络参数正确的分配给网域中的每部计算机， 让客户端的计算机可以在开机的时候就立即自动的设定好网络的参数值，使计算机可以顺利的连上Internet。

DHCP 通常是用于局域网络内的一个通讯协议，主要由客户端传送广播封包给整个物理网段内的所有主机，若局域网络内有 DHCP 服务器时，才会响应客户端的 IP 参数请求。所以

DHCP 服务器与客户端是要在同一个物理网段内

客户端搜索

DHCP服务器工作流程

得知这个客户端的硬件地址 (MAC) 与本身的数据来进行下列工作：

  1. 到服务器的登录文件中寻找该用户之前是否曾经用过某个 IP，若有且该 IP 目前无人使用，则返回此 IP给客户端
  2. 若配置文件对 MAC 提供了固定 IP (static IP) 时，则提供该固定 IP 给客户端
  3. 若不符合上述两个条件，则随机取用目前没有被使用的 IP 参数给客户端，并记录

整个流程

注意：

  1. 租约开始服务器开始计时
  2. 客户端脱机(关闭网口、重新启动、关机等),服务器回收ip，放入备用区，重开会重新走完整流程
  3. 租约到期，客户端自动重新申请，如果客户端没有重新申请，则会造成断网
