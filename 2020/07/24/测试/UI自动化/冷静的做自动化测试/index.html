冷静，在自动化和测试开发的路上要冷静，不要被看起来高大上又新鲜的框架技术洗白。

探索的历程

  1. Excel api test
  2. google sheet api test
  3. flask bootstrap4 api test & ui test
  4. pyqt5 ui test & api test % test tools
  5. vue element flask mysql ui test & ui test & test tools

冷静了，没有什么完美的平台和框架，找到其中一部分适合的，符合当前业务场景的即是最好的

多看坑

大佬们已经兜兜转转踩过的坑，要多去聆听下，不用自己再去踩一遍
UI自动化坑1

总结现在想到的完美组合

  1. UI自动化，只做类似表单提交有很多种组合的情况
  2. API自动化，尽量全部做完
  3. 业务常用重复劳动 vue ele flask mysql -> pyinstaller做成小工具即可
  4. 无线相关的也集成到小工具中

以上的百分比规划依赖公司测试任务量和测试人员配比情况
